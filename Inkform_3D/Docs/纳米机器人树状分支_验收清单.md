# 纳米机器人触手「树状分形分支」验收清单

> 关联需求：把触手伸出时的分支改为树状分形；每条分支导航到目标表面上各不相同的随机点；
> 接触表面后继续分叉、在表面分形蔓延（弱缠绕）；最终整表面均匀覆盖；分支不穿模；
> 所有路径在伸出前一次性解算好，分离后不回聚。

## 怎么验收

### A. 自动化（EditMode 单测）
Unity 菜单 `Window ▸ General ▸ Test Runner ▸ EditMode ▸ Run All`，`NanobotTreeTests` 应全绿：

| 测试 | 验收的需求 |
|------|-----------|
| `Build_OneLeafPerPoint_EndpointsAreInjective` | 每条分支导航到**各不相同**的表面点；叶子数 == 请求数 |
| `Build_AllPathsStartAtA` | 所有路径共享起点 A（统一树干根） |
| `Build_BranchesNeverReconvergeAfterDiverging` | **核心铁律**：任意两条路径一旦分叉，之后不再共享任何点（分离后不回聚） |
| `Build_IsDeterministic` | 同输入两次构树完全一致（伸出前已解算、确定性） |
| `Build_SurfaceSegmentsProjectedOntoSphere` | 贴面段投影回表面（不切入物体、贴面蔓延） |
| `BotPath_Sample_HitsStartAndLeaf_AndAdvances` | 路径采样起止正确、按弧长单调前进 |

### B. 手动（场景实测）
1. 打开 `Assets/Scenes/NanobotSandbox.unity`（或 `M1_TestScene`），进入 Play。
2. 按 `E` 扫描 → 选中可附身物体 → 鼠标左键确认。
3. 选中 `PossessionDirector` 物体看 Gizmos：绿色线段=分叉树、橙色点=叶子末梢，应在伸出前就成形且分布均匀。
4. 肉眼逐项核对：
   - [ ] 触手伸出即**空中树状分叉**（递归二分、自相似），非旧的横向支流。
   - [ ] 多条主分支分别奔向**不同的表面入口点**（非几何中心/单一点）。
   - [ ] 触面后在表面继续**分形分叉、蔓延铺开**，缠绕感弱、像蔓延。
   - [ ] 最终 `LeafCount` 个末梢在**整表面均匀分布**。
   - [ ] 全程分支**互不穿插**、不切入物体。
   - [ ] 分支分离后**不再回聚**（无"先散开又汇合"）。

### C. 可调参数（`PossessionDirector` Inspector）
`LeafCount`(覆盖分辨率) / `AirBranchDepth`(空中分叉级数) / `OutwardLift`(空中抬出高度) / `SurfaceSubdiv`(贴面细分) / `BranchThickness`(分支粗细) / `JitterAmp`(空中抖动) / `PathSpeed`(生长速度) / `WrapTriggerProgress`(上色触发) / `Trail`(队伍拉散)。
