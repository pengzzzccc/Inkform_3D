# Inkform — 3D 潜行解谜 GDD（垂直切片 / 单关 Demo）

> 文档定位：本文件描述 Inkform 的**单关垂直切片 Demo**，用于验证核心玩法。叙事为精简版本，规模可控；完整章节叙事另见 `Project_Prompt/`（旧 2D 方向，仅供参考，不作为本 Demo 依据）。

## 1. 高概念（High Concept）

Inkform 是一款 **3D 视角、电影化氛围的潜行解谜游戏**。玩家操控一个由墨水构成的生命体，从培养它的实验室逃脱。它能通过**扫描不同物品来获得对应能力**（扫描遥控器操控巨型机械、扫描船锚沉入水底……），并躲避周期性扫描的威胁、破解阻挡去路的谜题。

**一句话**：墨水之躯，遇形拟物，趋利避害，逃出生天。

### 设计支柱（Pillars）
1. **拟物变能**：每个能力既是潜行工具又是解谜钥匙，一物一解。
2. **静默潜行**：威胁是"环境压力"而非"战斗"，规避节奏制造紧张。
3. **环境叙事**：无对白，靠场景、光影、声音讲故事（INSIDE / Somerville 基调）。
4. **一物一思**：谜题围绕"当前该拟成什么形态"展开，鼓励观察与试验。

## 2. 故事与基调（精简）

- **设定**：一个由墨色纳米机器人构成的实验体在实验室深处苏醒，产生了逃离的意志。实验室仍在运转，自动化的扫描与清扫系统将一切异常视为需要清除的目标。
- **Demo 叙事弧**：苏醒（学会移动与拟物）→ 穿越被扫描封锁的区域 → 解开通往外界的出口谜题 → 在出口望向实验室之外的光。
- **基调**：压抑、孤独、好奇并存；冷色调、强光影；无台词，情绪靠节奏与声音传递。
- **主角表现**：由无数纳米机器人组成的可塑身体，基态为"半流体内核 + 硬表面板片外壳"的混合质感（深色半透明流体核心，外覆会流动重排的微小金属冷光板片）。扫描目标后，纳米机器人从初始状态**膨胀重组**为目标形态（参考《星之卡比 探索发现》塞满嘴变形的可爱与笨拙感），与冷峻环境形成反差。

## 3. 核心玩法循环

```
寻找线索 ──▶ 躲避扫描威胁 ──▶ 解谜推进 ──▶（前往下一段）
```

- **躲避段**：大型"清扫者"实体周期掠过 / 固定扫描装置规律扫描时，玩家必须躲入掩体或用能力规避，被光照到即死。高压、静默。
- **解谜段**：威胁过去或被规避后，玩家探索场景、寻找线索、扫描正确的物品拟成对应形态破解障碍。舒缓、思考。
- 两种段落**交替编排**，形成张弛节奏。

## 4. 操作与镜头

### 视角
- **电影化镜头**：以固定 / 轨道（rails）镜头为主，按场景切换机位，强调构图与氛围（参考 INSIDE 的 2.5D、Somerville 的固定机位）。
- **空间**：2.5D 纵深——玩家主要在一个平面内移动，局部有前后纵深通道。
- 镜头服务于"看清扫描光朝向与安全掩体"，关键威胁段保证可读性。

### 基础操作（始终可用）
| 输入 | 行为 |
|------|------|
| 移动 | 前后左右移动（基础行动力，拟成任何形态都保留；手感随形态变化） |
| 互动 | 触发机关、读取线索、扫描拟物 / 还原 |
| 使用能力 | 触发当前所拟形态赋予的能力 |
| 躲入掩体 | 贴靠/钻入场景掩体规避扫描 |

- **扫描拟物规则**：玩家一次只能模仿一种形态；需在扫描目标处扫描，可还原回基态再扫描别的。切换有空间成本（须回到对应物品处）。

## 5. 扫描拟物—能力系统（核心系统）

### 通用规则
- 一次只持有一种能力；扫描 = 获得能力并膨胀重组变身，还原 = 回到基础纳米形态。
- **移动玩法随所拟形态而变**：不同形态改变质量、速度、跳跃、浮力等（如船锚更沉、走更慢、跳更矮、可下沉；气球更轻、可飘浮）。
- 可扫描目标分布在场景中固定位置 → 形成"能力可达性"的空间约束，是关卡与谜题设计的杠杆。
- 变身有明确视觉/剪影差异，便于玩家与镜头读出当前状态。

### 能力清单（6 个）

**核心三能力（贯穿关键路径）**

| # | 扫描目标 | 效果 | 潜行用途 | 解谜用途 | 限制 |
|---|---------|------|---------|---------|------|
| 1 | **遥控器** | 远程操控场景中的巨型机械（吊臂、传送带、闸门、探照灯塔） | 关闭/转向扫描灯塔、移动巨型掩体挡住视线 | 移动平台搭桥、对齐齿轮/传送带运送物体 | 操控时自身须静止，期间易暴露 |
| 2 | **船锚** | 身体变重、可下沉 | 沉入水底通道躲过水面扫描 | 长按重量机关、抵抗气流/水流推力、沉到低处通道 | 变重后移动缓慢、无法跳跃/上浮 |
| 3 | **传送标记** | 放置一个标记，之后可随时瞬移回标记点 | 主动暴露自己把清扫者引向一侧，再瞬回安全标记 | 跨越单向断点、来回操作相隔两地的机关 | 同时只存在一个标记；瞬移有冷却 |

**段落专属能力（在特定段落引入，丰富谜题）**

| # | 扫描目标 | 效果 | 潜行用途 | 解谜用途 | 限制 |
|---|---------|------|---------|---------|------|
| 4 | **灯泡 / 电池** | 自身发光，或可向机关放电 | 照亮黑暗段（有暴露风险）/ 过载附近灯光制造短暂黑暗以躲扫描 | 为断电机关供电、激活光敏开关 | 发光会被扫描系统注意；电量/时长有限 |
| 5 | **磁铁 / 电磁体** | 带磁性，吸附金属 | 吸附一块金属板当**可移动掩体**挡住扫描光 | 拉动金属门、吸取金属钥匙、短接/激活电路 | 同时只能吸一块；负重移动慢 |
| 6 | **气球 / 氦气** | 变轻、可短暂飘浮上升 | 飘过地面压力感应区、升到高处掩体 | 越过高墙、够到高处开关 | 飘浮时间有限、易被气流吹动、落地有延迟 |

> **范围提示**：单关 Demo 中，遥控/船锚/传送承担关键路径与高潮段；灯泡·电池/磁铁/气球各负责一个独立小段落，避免一关内同时强塞 6 套教学。若制作压力大，可优先实现核心三能力 + 任一段落能力作为最小可玩版本。

## 6. 威胁与潜行系统

### 死亡判定
- **被扫描红光 / 探照光照到 = 即死**，立即返回最近的分布式检查点。
- 无血条、无战斗血量，威胁是二元的（安全 / 死亡），强化潜行张力。

### 混合威胁来源
1. **固定扫描装置**：旋转探照灯、规律横扫的激光栅、定点摄像头。有可预测的扫描周期，玩家须卡节奏穿过或用能力规避。
2. **大型"清扫者"实体**：一个周期性掠过区域的庞然巨物（参考《风之旅人》隧道龙、《萨默维尔》外星飞船）。
   - 行为循环：**远处逼近 → 进入并扫描区域 → 离开**。
   - 扫描期间玩家必须藏入掩体 / 用能力规避（磁铁掩体、船锚下潜、传送引开、气球避到高处死角）。
   - 是 Demo 的标志性高压来源与高潮段落。
3. **（可选）移动巡逻者**：无人机/保安类，**面朝方向散发红光，照到即死**；其**实际感知范围大于红光范围**（红光是死亡判定，感知用于触发警觉/转向）。

### 规避手段
- **场景掩体**：箱体、立柱、阴影区、可钻入的管道。
- **能力规避**：磁铁拖动金属板做移动掩体；船锚潜入水下；气球升到高处；传送标记把自己引开后瞬回。
- **节奏规避**：观察扫描周期，在间隙中移动。

### 检查点
- **分布式检查点**（参考 INSIDE）：关卡内多个隐式检查点，死亡回到最近一处，节奏友好、鼓励试错。

## 7. 谜题设计

### 谜题类型与能力钩子
| 谜题类型 | 对应能力 | 示例 |
|---------|---------|------|
| 机械操控 | 遥控器 | 操控吊臂把集装箱搭成桥；让传送带把钥匙运到可达处 |
| 重量 / 水位 | 船锚 | 沉入水底打开水下阀门；用重量长压机关固定门开启 |
| 空间错位 | 传送标记 | 在 A 处启动机关后瞬回 B 处趁机通过单向门 |
| 电路 / 供电 | 灯泡·电池、磁铁 | 给断电的门供电；短接两个金属触点接通电路 |
| 光敏 / 黑暗 | 灯泡·电池 | 照亮光敏开关；制造黑暗让光感扫描失效 |
| 高度 / 触达 | 气球 | 飘到高处按下被隔开的开关 |

### Demo 核心谜题（建议 3 个）
1. **水闸谜题（船锚）**：被淹没的通道阻路，扫描船锚拟成其形态、沉到水底打开排水阀，水位下降后腾出新路径。
2. **吊臂搭桥谜题（遥控器 + 固定扫描）**：用遥控器操控吊臂移动集装箱搭桥，同时躲避旋转探照灯——操控时自身静止，须算准灯的扫描间隙。
3. **断点错位谜题（传送标记 + 清扫者高潮）**：在清扫者周期掠过的大厅中，先放标记于安全死角，再趁间隙冲到远端启动出口机关，于扫描临近前瞬回标记。

## 8. 单关流程（Demo Walkthrough）

| 段落 | 内容 | 引入能力 | 主要威胁 | 目的 |
|------|------|---------|---------|------|
| A. 苏醒区 | 学习移动、扫描拟物/还原、躲入掩体 | （基础） | 无 / 轻度 | 教学，建立基调 |
| B. 扫描走廊 | 卡固定探照灯节奏穿过 | — | 固定扫描装置 | 教"观察周期、利用掩体" |
| C. 水区 | 水闸谜题 | **船锚** | 水面扫描 | 教扫描拟物-解谜闭环 |
| D. 机械大厅 | 吊臂搭桥谜题；磁铁移动掩体小段 | **遥控器**、**磁铁** | 固定扫描 + 旋转灯塔 | 复合能力运用 |
| E. 黑暗供电段 | 给出口前置门供电/光敏开关 | **灯泡·电池** | 黑暗中的局部扫描 | 风险收益抉择 |
| F. 清扫者高潮 | 大厅断点错位谜题 | **传送标记**（+气球避高处死角） | **大型清扫者掠过** | 高压综合关 |
| G. 出口 | 解最后机关，门开 | 综合 | 收束 | 释放紧张 |
| H. 结尾镜头 | 墨水生命体走出，望向外界的光 | — | — | 情绪收尾、留钩子 |

> 顺序可调整；气球段可并入 F 或单独成段（如够取高处的传送物品）。

## 9. 关卡元素清单（制作清单）
- **可扫描目标点** ×6（遥控器、船锚、传送标记、灯泡/电池、磁铁、气球）——固定刷新位。
- **掩体**：静态掩体、可钻管道、阴影区。
- **威胁体**：固定探照灯/激光栅/摄像头、清扫者实体、（可选）巡逻无人机。
- **机关**：水闸阀、吊臂/传送带、重量板、供电点/光敏开关、单向门、出口门。
- **分布式检查点** ×N。
- **线索物**：环境提示（标识、痕迹、灯光指引），无文字或极少文字。

## 10. UI / HUD

- **极简，趋近无 HUD**（参考 INSIDE）。
- 仅保留：**当前能力图标**（角落小图标 / 或纯靠变身剪影表达）、**互动提示**（靠近可交互物时的微弱高亮）、**死亡淡入淡出**。
- 无血条、无小地图、无任务文字；引导靠关卡设计与光影。

## 11. 美术与音频方向

### 美术
- 低饱和冷色调、强光影对比、剪影化构图；扫描光/探照光是画面中少数高亮元素，天然引导注意力。
- 角色：半透明墨水质感、可塑形；变身时身体被物品"撑变形"，保留墨水流动感。

### 音频
- 环境音主导，强空间感；几乎无配乐或极简配乐。
- **清扫者**有标志性声音线索（低频逼近声）作为玩家的"听觉雷达"。
- 被发现瞬间：急促、刺耳的反馈音 + 镜头/画面冲击。

## 12. 技术与范围（Unity）

- **引擎**：Unity（沿用项目现有版本）。
- **架构复用**：现有项目（虽为 2D）已有一批可复用的**架构模式**，3D 化时应沿用其思想而非重写：
  - 事件总线 `S_GameEvent`（解耦威胁/机关/检查点广播）。
  - `ManagerRoot` 跨场景持久化、输入锁模式。
  - 关卡分段（Sections）、关卡配置 `S_LevelConfig`。
  - NPC 传感器 / 巡逻-警觉状态机（用于巡逻者与扫描判定的 3D 改造）。
  - 检查点 / 进度（Progression）系统。
  > 注意：上述实现当前面向 2D，需评估 3D 化改造成本；本 Demo 视为新方向的技术验证。
- **Demo 范围 / 里程碑（建议）**：
  1. M1：基础移动 + 扫描拟物/还原 + 一个固定扫描威胁 + 死亡/检查点（段落 A、B）。
  2. M2：核心三能力（遥控/船锚/传送）+ 对应谜题（段落 C、D、F）。
  3. M3：段落专属能力（灯泡·电池/磁铁/气球）+ 美术音频氛围 + 结尾镜头（段落 E、G、H）。
